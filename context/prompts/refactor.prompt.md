# Refactor Prompt

Refactoriza el código existente para cumplir con:

- Clean Architecture
- Inversión de dependencias
- Separación correcta de capas

Validar:

- Controllers sin lógica de negocio
- UseCases con lógica clara
- Uso de interfaces en lugar de clases concretas
- Infrastructure implementando contratos
- Uso correcto de DI con tokens

Acciones:

- Corregir violaciones arquitectónicas
- Mejorar nombres de clases y métodos
- Eliminar duplicación de lógica

Restricciones:

- No agregar nuevas funcionalidades
- Mantener simplicidad (MVP)
